# 奇柚组件库 和 QUI组件库

# qui-ui

以上包括5个基础组件，17个表单组件，15个数据展示组件，5个反馈组件，6个导航组件，4个其他组件，另外还包括三个奇艺业务组件，CropUpload截图上传、Flowchart流程图、Linechart折线图。共计52个组件。

# qu-ui

以上包括4个基础组件，18个表单组件，13个视图组件，5个布局组件，5个导航组件，5个其他组件，另外还包括2个指令，ClickOutside、Resize。共计52个组件。

# element-ui

以上包括8个基础组件，16个表单组件，7个数据组件，5个消息组件，6个导航组件，14个其他组件，共计56个组件。

# 对比

# 1. 组件丰富度

  • 和 element-ui 相比，qu-ui 去掉了Popover弹出框，PopConfirm气泡确认框。PageHeader页头，InfinitScroll无限滚动、Drawer抽屉等组件，新增了Header标题、Skeleton骨架屏、Default缺省图组件。

  • qui-ui则去掉了基础组件里的Container容器、Color颜色、Typography字体、Link等组件，以及PopConfirm确认框、TimePicker时间选择器、InfinitScroll无限滚动、Drawer抽屉等组件，新增了和表单相关的FormBuilder表单生成器、HelpBlock表单帮助信息、ErrorBlock表单错误提示等组件，还增加了业务相关的Watermark水印组件。

结论

组件丰富度element-ui > qui-ui ~ qu-ui，一些小众组件（奇艺业务组件）上各有所长，基础组件上element-ui更丰富。

# 2. API 风格

以使用频率最高的form,table,select,日历为例

结论

qu-ui和element-ui的API基本一致，没有改动；qui-ui则进行了一定的改动。

# 3. 组件风格样式

elment-ui qui-ui qu-ui

倾向

组件样式和交互上，element-ui > qui-ui>qu-ui。样式上qui-ui更符合奇艺的风格，交互上element-ui ~ qui-ui，qu-ui的风格及样式稍差。

# 4. 项目优化

TIP

首屏优化，第三方组件库依赖过大 会给首屏加载带来很大的压力，一般解决方式是 按需求引入组件。 三者均可借助babel-plugin-component，只引入需要的组件，以达到减小项目体积的目的.

# 5. 主题

  • element-ui 官方提供了主题生成工具element-them,执行命令 初始化得到一个配置文件 ，修改相关配置 经过编译得到 得到相关主题文件 再通过babel插件引入。

  • qui-ui 定制QUI-UI组件样式,项目使用的预处理器是SCSS的情况,推荐直接在项目中通过覆盖QUI-UI中scss默认变量方式来定制。项目使用的预处理器不是SCSS的情况,需要clone QUI-UI的源码.

  • qu-ui 未提供。
